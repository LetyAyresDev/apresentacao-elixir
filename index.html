Por que tratar erros?

Todo sistema está sujeito a falhas.

O importante não é evitar erros, mas tratá-los corretamente.

Elixir foi criado com foco em tolerância a falhas.

O que é Elixir?

  • Linguagem funcional moderna
  • Executada na máquina virtual Erlang (BEAM)
  • Alta concorrência
  • Escalabilidade e robustez

Filosofia do Elixir

Elixir prefere retornos previsíveis ao invés de falhas inesperadas.

{:ok, resultado}

{:error, motivo}

Como o Elixir trata erros?

Em muitas linguagens, um erro pode interromper o sistema.

No Elixir, o retorno normalmente é controlado. 


{:ok, valor}
{:error, motivo}

Entendendo as Tuplas

As tuplas armazenam múltiplos valores.

O primeiro informa o status.

O segundo contém o resultado ou erro. 


{:ok, 10}
{:error, "Falha"}

Fluxo do Tratamento

Usuário digita valor
↓
Função recebe entrada
↓
Sistema verifica erro
↓
{:ok, resultado}
ou
{:error, motivo}

Exemplo Visual

10 / 0
↓
Sistema identifica erro
↓
{:error, "Divisão por zero"}

Sistema continua funcionando

Exemplo de tratamento 


def dividir(a, b) do

  if b == 0 do
    {:error, "Divisão por zero"}

  else
    {:ok, a / b}

  end

end

Uso do case 


case dividir(10, 0) do

  {:ok, resultado} ->
    IO.puts(resultado)

  {:error, motivo} ->
    IO.puts(motivo)

end

Try / Rescue 


try do

  10 / 0

rescue

  ArithmeticError ->
    IO.puts("Erro matemático")

end

O que é Escript?

Ferramenta para gerar executáveis.

Permite executar aplicações no terminal.

Facilita distribuição do sistema.

Gerando Executável 


mix escript.build

Conclusão

Elixir possui tratamento de erros eficiente.

O escript facilita a criação de executáveis.

A linguagem prioriza estabilidade e confiabilidade.
